Descripción del Proyecto

Este proyecto desarrolla un sistema basado en visión artificial y control de hardware que permite detectar frutas no maduras en tiempo real utilizando una Raspberry Pi 3 Model B+. La detección se realiza a través de procesamiento de imágenes, identificando colores específicos que indican inmadurez (como tonos verdes). El sistema emplea dos LEDs como indicadores visuales del estado de detección:

-LED 1 se enciende cuando se detectan frutas no maduras.

-LED 2 se enciende cuando no se detectan frutas no maduras.

El diseño busca integrar tecnologías de bajo costo con técnicas avanzadas de procesamiento de imágenes, creando una solución accesible y educativa que combina la electrónica básica con la programación y la inteligencia artificial aplicada.

Objetivos :

-Detección precisa de frutas maduras y no maduras.

-Implementar los LEDs que proporcione una retroalimentación inmediata.

-Facilidad de uso e implementacion.

Marco teorico:

La visión por computadora es una rama de la inteligencia artificial que permite a las máquinas interpretar y analizar imágenes digitales. En este proyecto, se utiliza el procesamiento de imágenes para segmentar colores específicos y detectar objetos en tiempo real. El espacio de color HSV es especialmente adecuado para este propósito porque separa el matiz (Hue), la saturación (Saturation) y el brillo (Value), lo que facilita identificar colores independientemente de variaciones en la iluminación. Esto es crucial en aplicaciones como la detección de frutas, donde los colores de los objetos son un indicador clave de su estado.

La segmentación de colores implica aislar regiones de una imagen que comparten características de color específicas. Este proceso comienza con la conversión de la imagen al espacio HSV, seguido de la creación de una máscara binaria que separa los píxeles relevantes (en este caso, verdes) del resto. Una vez creada la máscara, se detectan los contornos para identificar las formas de los objetos dentro de las regiones segmentadas. Los contornos son útiles para calcular propiedades geométricas como el centroide, el área y la orientación de los objetos.

La Raspberry Pi actúa como el núcleo del sistema, combinando la capacidad de procesar imágenes con el control de hardware externo, como LEDs. Los LEDs proporcionan una interfaz de usuario intuitiva basada en principios de retroalimentación visual. En un sistema binario como este, el encendido de un LED específico indica el estado del sistema (fruta detectada o no detectada), lo que es un ejemplo simple pero eficaz de interacción entre software y hardware.

Los pines GPIO de la Raspberry Pi son una herramienta poderosa para interactuar con componentes electrónicos. En este proyecto, los GPIO controlan LEDs que actúan como indicadores visuales. Los pines pueden configurarse para enviar señales digitales (HIGH o LOW), lo que permite activar o desactivar dispositivos externos. Este diseño refleja principios básicos de sistemas embebidos, donde el software controla hardware físico para realizar tareas específicas.

Metodologia:

1.Configuración de la Raspberry Pi para capturar video en tiempo real mediante una cámara conectada.

2.Conexión de los LEDs a los pines GPIO, asignando cada LED a un estado específico.

3.Procesamiento de Imágenes.

4.Captura de video con OpenCV.

5.Conversión de cada fotograma al espacio de color HSV, que permite identificar rangos de color con mayor precisión.

6.Aplicación de un rango de color predefinido para generar una máscara binaria que destaque las áreas verdes.

7.Análizar de contornos para localizar frutas no maduras.

8.Si se detecta si la fruta esta madura prendera su respectivo led.

Concluciones:

El proyecto logra su objetivo principal de identificar frutas no maduras y proporcionar retroalimentación visual clara mediante LEDs. La arquitectura del sistema es modular, permitiendo su ampliación con algoritmos más avanzados o sensores adicionales para un análisis más completo de las frutas.
